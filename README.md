# PDN Scanner Solution

Программа для поиска утечек персональных данных (ПДн) в файловом хранилище. Сканирует различные форматы файлов, извлекает текстовое содержимое (в том числе через OCR для изображений и видео) и определяет наличие персональных данных по регулярным выражениям и контекстному анализу.

## Общая информация

### Цель
Обнаружение файлов, содержащих потенциальные утечки персональных данных.

### Ключевые возможности
- Рекурсивный поиск по директории `dataset/`
- Автоматическое определение типа файла
- Извлечение текста из документов, таблиц, изображений и видео
- Поиск 15+ типов персональных данных
- Контекстный анализ (поиск по названиям полей и ключевым словам)
- Валидация данных (проверка контрольных сумм для СНИЛС, ИНН, банковских карт)
- Параллельная обработка больших наборов файлов
- Фильтрация ложных срабатываний (шаблоны, примеры, формы)

---

## Поддерживаемые форматы

### Документы и таблицы
- **CSV** — табличные данные (поддерживаются разные разделители и кодировки)
- **JSON** — структурированные данные с анализом имён ключей
- **Parquet** — бинарный табличный формат
- **PDF** — извлечение текста со всех страниц
- **DOCX** — параграфы и таблицы из документов Word
- **HTML** — очистка скриптов/стилей, извлечение текста

### Мультимедиа
- **Изображения** — TIF, TIFF, JPG, JPEG, PNG, GIF, BMP (распознавание через OCR)
- **Видео** — MP4, AVI, MOV, MKV (извлечение и анализ кадров по интервалу)

### Fallback
- Любые текстовые файлы через универсальный экстрактор

```
│   └── factory.py        — фабрика для создания экстракторов
├── detectors/           — поиск и валидация ПДн
│   ├── scanner.py        — основная логика детектирования

│   └── classifier.py     — классификация типов ПДн
└── reports/result_writer.py — сохранение результатов в CSV
```

### Поток обработки

```
1. Инициализация
   └─ Проверка папки dataset

2. Обход файлов
   └─ scanner/discovery.py :: iter_files()
      └─ Рекурсивный поиск по расширениям

3. Фильтрация
   └─ Исключение файлов < 100 байт
   └─ Разделение на текстовые и медиа

4. Параллельная/Последовательная обработка
   └─ process_tasks() → process_tasks_parallel() или process_tasks_sequential()

5. Для каждого файла
   ├─ Определить тип (FileKind)
   ├─ Получить экстрактор из кэша (thread-safe)
   ├─ extract() — извлечь текст и метаданные
   ├─ detect_pdn() — поиск ПДн
   ├─ has_personal_data() — проверка критериев
   ├─ _is_real_data_leak() — фильтрация шаблонов
   └─ Добавить в результаты (если подходит)

6. Сохранение
   ├─ result_text.csv (текстовые файлы)
   ├─ result_video.csv (изображения/видео)
   └─ result.csv (объединённый результат)
```

## Установка

### Требования
- Python 3.10+
- pip

### Шаги установки

1. **Создать виртуальное окружение** (опционально)
```bash
python -m venv .venv
source .venv/Scripts/activate  # Windows
# или
source .venv/bin/activate      # Linux/Mac
```

2. **Установить зависимости**
```bash
pip install -r requirements.txt
```

Или вручную:
```bash
pip install pandas pyarrow pypdf python-docx beautifulsoup4 pillow opencv-python easyocr openpyxl
```

3. **Проверить установку**
```bash
python -m py_compile main.py detectors/scanner.py extractor/*.py
```

## Детектирование ПДн

### Типы обнаруживаемых данных

#### 1. Общие идентификаторы
- **Email** — стандартный формат `user@domain.com`
- **Телефон** — номера вида `+7 (XXX) XXX-XX-XX`, `8 XXX XXX XX XX`
- **Дата рождения** — формат `DD.MM.YYYY`, `DD/MM/YYYY`, `DD-MM-YYYY`
- **ФИО** — фамилия, имя, отчество (русский и английский)

#### 2. Государственные идентификаторы
- **Паспорт РФ** — формат `XX XX XXXXXX`
- **СНИЛС** — формат `XXX-XXX-XXX XX` (с валидацией контрольной суммы)
- **ИНН** — 10 или 12 цифр для ЮЛ/ФЛ (с валидацией)
- **MRZ** — Machine Readable Zone из документов

#### 3. Платёжные данные
- **Банковская карта** — 13-19 цифр (проверка алгоритма Луна)
- **БИК** — 9-значный код (с валидацией)
- **Расчётный счёт** — 20 цифр
- **CVV/CVC** — 3-4 цифры после номера карты

#### 4. Специальные категории
- **Здоровье** — диагнозы, болезни, пациенты, инвалидность
- **Религия** — конфессия, вероисповедание
- **Политика** — политические убеждения, партия
- **Раса/Национальность** — этническая принадлежность

#### 5. Биометрия
- **Биометрические данные** — отпечатки, радужка, лицо, голос

### Критерии обнаружения утечки

Файл считается содержащим ПДн, если выполняется одно из условий:

1. **2+ сильных совпадения** (email, телефон, паспорт, СНИЛС, ИНН, MRZ, карта, CVV)
2. **1 идентификатор + дополнительные признаки** (государственный документ + адрес/рождение)
3. **2+ email/телефона** (даже без других данных)
4. **Спецкатегория** (здоровье, религия, политика, раса)
5. **Биометрия** (любое совпадение)
6. **4+ мягких совпадения** (ФИО, контекст адреса)

### Фильтрация ложных срабатываний

Программа отклоняет файлы, если:

- **Ключевые слова формы** — "анкета", "форма", "шаблон", "пример", "бланк"
- **Только одно ФИО без контекста** — игнорируется
- **CSV/JSON без контекста в заголовках** — номера без явного назначения
- **Табличные данные (CSV/JSON/Parquet)**:
  - Номер требует контекста в имени колонки или 4+ совпадения вместо 2

### Контекстный анализ

Для повышения точности программа проверяет:

- **Имена полей в таблицах** — "passport", "паспорт", "card_number" и т.д.
- **Ключи в JSON** — структура данных подтверждает назначение
- **Ключевые слова в тексте** — "паспорт РФ", "банковская карта" и т.д.

## Результаты

### Формат выходных файлов

Все результаты сохраняются в формате CSV:
```
size,time,name
```

- **size** — размер файла в байтах
- **time** — время последнего изменения (формат: `mon dd hh:mm`)
- **name** — имя файла

### Файлы результатов

1. **output/result_text.csv** — документы, таблицы, PDF, HTML
2. **output/result_video.csv** — изображения и видео
3. **output/result.csv** — объединённый итоговый отчёт (удалены дубликаты, отсортированы)

### Примеры обнаруженных файлов

```
size,time,name
3068287,sep 26 18:31,CA01_01.tif
4265542,sep 26 22:01,client.parquet
8879363,sep 26 22:01,physical.parquet
1026771,sep 26 14:25,Координаты_для_связи.pdf
11338525,sep 26 18:40,CS10.mp4
```

## Оптимизация производительности

### Многопоточная обработка
- По умолчанию для 50+ файлов используется `ThreadPoolExecutor`
- Количество потоков: `min(4, len(tasks) // 20 + 1)` (адаптивное)
- Кэширование экстракторов на поток (избегаем race conditions)


## Примечания

### О лицензировании
Используются открытые библиотеки:
- `pandas`, `openpyxl` — работа с табличными данными
- `pypdf` — PDF
- `python-docx` — DOCX
- `beautifulsoup4` — HTML
- `pillow`, `opencv-python` — обработка изображений
- `easyocr` — распознавание текста

### Ограничения
- Для больших видео файлов (> 1GB) обработка может быть долгой
- OCR зависит от качества исходного изображения
- Текст на сложных фонах или со специальными эффектами может распознаваться неточно
